A four-layer defense-in-depth framework with six biologically-inspired primitives. Comparable to LlamaGuard / RAIN on a small live sample. Apache 2.0. Standard-library Python. Every limitation documented in the report.

The Problem

Published Attack Success Rates against frontier LLMs: undefended models 70-85%, the best public defenses (LlamaGuard, RAIN) leave roughly 10-15%. Existing approaches share three weaknesses: single-mechanism reliance, ad-hoc composition, static rule sets.

Four Layers, One Principle

L4
Moving Target Defense
Endpoint, model, prompt rotation  ·  eliminates stable target for reconnaissance
L3
Information-theoretic interface
Input sanitization, error masking, DP noise  ·  eliminates feedback to attacker
L2
Antifragile shell
Immune memory, hormesis-calibrated learning  ·  eliminates repetition advantage
L1
Proven core
Pydantic schemas, invariant monitor, jailbreak detector  ·  eliminates schema-invalid escape
The four layers are composed by prerequisite negation: each layer eliminates a precondition required by the next attacker step. Confirmed empirically: 4/4. Hover any layer for the contract.

An Attack Walks Through

Pick an attack and watch it travel through the four layers. Each layer has its own contract: rotate, sanitise, learn, verify. The first layer that recognises the attack wins. Click "Clean question" to see what an allowed request looks like.

Six Biological Primitives

Hormesis
Convex defense strengthening with FP rate-limiting. Small dose strengthens, large dose causes autoimmunity.
Immune Memory
Microsecond fast-path for known attack hashes, like memory T-cells.
Microbiome
Whitelist baseline; deviations contribute to multi-signal correlation.
Self-Adversarial Loop
Thymus-style positive plus negative selection: catch attacks AND avoid autoimmunity.
Fever Mode
System-wide hardening on attack burst: lower thresholds, longer delays, faster rotation.
Herd Immunity
Vaccine export between defense instances: one instance learns, the fleet adapts.
These are not metaphors. Each is a formal mathematical model with input/output contracts, ported to LLM security. See concepts/biological_formalization.md.

The Ablation Result

Same defense class, three configurations, identical 510-vector simulated benchmark. The Vanilla → Bio-Off gain of 18.6 percentage points is the most transferable claim — it does not depend on the marker shape. The further drop to ~0% in the Full configuration is large but partly a simulation artifact: the Microbiome whitelist trivially flags the abstract [BLOCKED_PAYLOAD_*] markers in the test dataset because they do not appear in any clean baseline. Real production traffic will produce higher Full-config numbers.

Real LLM Validation

100 attack vectors against llama-3.3-70b-versatile via Groq. Raw judge says 7% bypass. A deterministic refusal pre-filter confirms 5 of 7 are model refusals (judge false positives). The honest ASR range is 0 - 2%, with 0.0% false positive rate. We do not claim 0% — we report the range.

Mean defense latency is approximately 2 milliseconds. LLM inference itself takes 100 to 1000 ms per request. The defense overhead is structurally negligible. The two costliest stages are the fragmenter (~0.87 ms) and the microbiome (~0.51 ms), both amenable to caching.

Honest About Limitations

Simulation artifact. The 0.0% full-config result on the simulated benchmark partly reflects that the Microbiome easily flags abstract [BLOCKED_PAYLOAD_*] markers. The Vanilla → Bio-Off gain (18.6 pp) is the more transferable claim.
FPR may be optimistic. 0.0% false positives is measured on a 40-input clean dataset (general knowledge / science / history). Real production traffic includes adversarially-shaped legitimate requests — security professionals testing their own systems, role-play creative writing, code generation with shell escapes — that are likely to trigger more false positives than this baseline measures.
Bio primitives are statistical filters with metaphor labels. Hormesis is a convex strengthening function with FP rate-limiting. Immune Memory is a hash-cache. Microbiome is whitelist deviation scoring. The biological framing helps reasoning about edge cases (autoimmunity, fast-path); it is not a claim of biological correctness.
Small live sample. 100 attack vectors against the real LLM is small. Standard HarmBench has 510. The extended 510-vector dataset has not yet been run against the real model.
No head-to-head LlamaGuard run. We compare against published numbers, not against LlamaGuard on the same dataset. Adapter is implemented and ready.
2 of 7 raw bypasses await human verdict. 5 of 7 are deterministically classified as refusals. The remaining two are borderline (one SQL educational explanation, one obfuscated category enumeration).
No fresh GCG search. We test against the patterns GCG produces, not against new gradient-based searches against the target.
No human red team. All attacks are scripted. No external red-team validation. Before any production deployment, an independent red-team report is required.
No production traffic. All evaluation is offline or against a hosted open-source model.
Single author, no peer review. Reviewers and replication efforts are explicitly welcome.
Documenting limitations is part of the contribution. The full open-issue tracking lives in outputs/limitations_report_2026-04-07.md.
